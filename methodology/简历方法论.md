# 简历方法论

> 本项目所有简历改写、JD 匹配、邮件生成都遵守本文件。配套文件：《[风格库](风格库.md)》（企业性质写法）、《[岗位速查](岗位速查.md)》（岗位专业词）、《[挖掘问题库](挖掘问题库.md)》（提问清单）。
> 来源说明：方法论框架受 [industry-resume-toolkit](https://github.com/shangsitongshizaitiantang/industry-resume-toolkit)（CC BY-NC-ND 4.0）与 [Resume-Tailor-AI](https://github.com/JaimeYeung/Resume-Tailor-AI) 启发，全部内容为本项目原创提炼表述，未复制原文。

## 三条铁律

1. **保真**：简历上每个字都要经得起面试追问。判断标准只有一个——被追问三层还不露馅才能写。数字必须来自 profile/ 事实清单；缺数据就问本人，本人没确认就不写，绝不推测补数。职责可以重新组织语言，不能升格（实习生 ≠ 负责人）。
2. **一页 A4**：硬约束。每次生成或修改简历后必须跑 `scripts/render.ps1` 验证页数，超页就删减内容，不允许用缩小字号（< 10pt）或压行距蒙混。
3. **价值翻译**：简历是写给 HR 和用人经理看的，不是工作日志。每条经历回答的问题是「我为什么对你有用」，不是「我干过什么」。

## 改写三步

### 1. 排雷（信息卫生）

- 模糊量词清零：「很多、大量、丰富、多次、诸多」→ 换成具体数字，拿不到数字就换写法
- AI 味虚词清零：「赋能、打造、深耕、多维度、无缝衔接、抓手」不出现
- 每条以动词开头，不写「我」；语气克制，形容词能删就删
- 备考中的证书不写；已过部分科目写「XX（已通过 N 科）」
- Office 不拆成 Word/Excel/PPT 单列；真正的差异化工具（Wind、iFinD、Python）单独写
- 成绩要么成体系放，要么不放，不挑高分放
- 联系方式一行解决，不放二维码、家庭住址、多个邮箱

### 2. 价值翻译

- 公式：**动作 + 落地方式 + 量化结果**。三要素缺一就是半成品 bullet
- 动作 → 业务意义：「整理数据」→「维护 XX 数据库，支撑 X 期复盘汇报」
- 量化维度任选其一：数量 / 频率 / 规模 / 结果 / 名次 / 覆盖面
- 关键词对齐：用 JD 里反复出现的动词和名词改写表述（对齐 ≠ 堆砌，一条 bullet 最多命中 1-2 个关键词）
- 关联度排序：与 JD 强相关的经历放前、写满；弱相关的压成一行或删除

### 3. 深挖（信息不够时向本人提问）

六个维度，一轮最多问 3-5 个，只问事实、不问动机和感受：

1. 业务背景 —— 这是什么业务？服务谁？
2. 规模数据 —— 处理了多少条 / 多少篇 / 多少人 / 多少钱？
3. 个人贡献 —— 你（不是团队）具体做了什么？
4. 成果 —— 结果如何？有没有数字或者反馈？
5. 困难与解决 —— 卡在哪里？怎么解决的？
6. 关键判断 —— 你自己做过什么决定？

答案一律回写进 `profile/experiences/` 对应文件的事实清单。按经历类型的具体追问角度和「全面普查」清单见《[挖掘问题库](挖掘问题库.md)》。

## JD 分析框架

拿到 JD 先提取七类信息（软技能词如「沟通、团队合作」不算关键词，不提取）：

| 类别 | 提取内容 |
|---|---|
| 硬性门槛 | 学历 / 专业 / 年级 / 证书 / 每周天数与持续时长 / 必备工具 —— 逐条比对，不满足要明示风险 |
| **文件命名要求** | **JD 中明确要求的简历文件命名格式**（如「姓名-学校-专业.pdf」「学校+姓名+应聘岗位」等）—— 没有要求时默认 `城市-学校-姓名-专业-年级` |
| 职能关键词 | 岗位名及其变体，判断这个岗位属于哪个职能族 |
| 硬技能 | 工具、软件、方法名 —— 能被机器筛精确匹配的词 |
| 动作动词 | 职责描述里反复出现的动词（跟踪 / 搭建 / 输出 / 落地…） |
| 领域术语 | 行业黑话与业务语境词（产业链环节、指标名、品类名） |
| 加分项与风格信号 | 标「优先 / 加分」的条目；企业性质与文化用语 → 对应《风格库》选节 |

**命名要求常见表述**：「请以"姓名-学校"命名」「简历命名格式：学校+专业+姓名」「附件命名：姓名_学校_岗位」「文件名请注明应聘岗位」等。第一次遇到某公司的命名要求时，记入《岗位速查》对应公司章节。

按出现频次和位置（标题 > 职责首条 > 任职要求）排出 **Top 10 关键词**，再把本人经历对着 Top 10 分三类（区分后两者是关键）：

- **已覆盖**：简历直接能证明的
- **可挖**：做过但没写出来 / 表达不到位的 —— 靠深挖和改写解决
- **真缺失**：没做过的 —— 保真铁律，不编，考虑用相邻能力弱化处理

## 企业性质与岗位

写法细则见《[风格库](风格库.md)》（券商金融 / 大厂 / 国央企 / 外企四节）和《[岗位速查](岗位速查.md)》（行研 / 数据分析 / 产品）。一句话版：金融要严谨精准，大厂要数字与 JD 语言，国央企要完整稳重，外企要 STAR 与强动词。

## 一页纸内容预算（默认模板下）

| 区块 | 预算 |
|---|---|
| 页眉（姓名+联系） | 2 行 |
| 教育背景 | 3-4 行 |
| 实习经历 | 每段 3-4 条 bullet，每条 ≤ 2 行 |
| 实践与竞赛 | 3-4 个条目，每个 1-2 行 bullet |
| 技能与证书 | 3-4 行 |
| 补充信息（可选） | 1 行（到岗时间等） |

超页删减顺序：弱相关竞赛 → 校园工作 → 爱好 → 压缩 bullet 第二行 → 与本人商量。

## 投递邮件规范

> **核心原则**：HR 先看邮件正文，再决定要不要打开简历。正文必须真诚、有针对性，让人看出「你是专门为这个岗位准备的」，而不是批量海投。

### 邮件主题

- **JD 指定格式优先**（如有要求，一律按 JD）
- 默认格式：`应聘【岗位全称】- 学校 - 姓名 - 每周X天 - 可实习X个月`
- 关键信息前置：实习天数和时长是硬性筛选条件，写主题里省得 HR 还要打开看

### 正文结构（3-4 段，250-350 字）

**第一段：开场（2-3 句）**
- 我是谁（学校+专业+年级）
- 从哪里看到这个岗位（官网/内推/熟人推荐）
- 为什么投这个岗位（**不是「贵司平台好」，而是岗位职责和你的方向匹配**）

**第二段：匹配点（3-4 句，核心段落）**
- **换位思考**：JD 里 HR 最担心什么？（经验不足？工具不会？时间不够？）针对性回应
- **用具体事实证明匹配**，不要写「我具备XX能力」，而是「我在XX经历中做过XX，产出了XX」
- **对齐 JD 硬性要求**：
  - JD 要「熟悉 Python」→ 写你用 Python 处理过多少数据、做过什么分析
  - JD 要「每周 4 天」→ 明确说「可以每周 X 天，持续到 X 月」
  - JD 要「3 个月以上」→ 说清楚你的时间安排（考试周可能请假 1 周，寒假回校等）
- **体现你研究过公司**：
  - 提到公司近期动态（如刚发布的产品/报告/融资消息）
  - 或者你为什么对这个行业/赛道感兴趣（但要真实，不要编）

**第三段：作品集/GitHub（如有）**
- 如果 `profile/links.md` 中有匹配的链接（标注「适用场景」匹配当前岗位），放这里
- 一句话说明链接内容（如「附上我做的 XX 数据分析项目，用 Python 实现了 XX 功能」）

**第四段：到岗时间 + 附件 + 收尾（2 句）**
- **到岗时间具体写**：「X 月 X 日可以到岗」，不要写「随时」（显得不走心）
- 附件说明：「简历见附件（已按贵司要求命名为 XX.pdf）」
- 收尾：「期待您的回复」或「感谢您的时间」（简洁即可，不要堆砌客套话）

### 写作红线（以下表述禁止出现）

❌ **批量海投痕迹**：
- 「贵司平台好」「行业领先」「学习机会多」—— 任何公司都能套
- 「我对 XX 行业很感兴趣」但说不出具体为什么

❌ **空洞自夸**：
- 「我学习能力强」「沟通能力好」「抗压能力强」—— 没有事实支撑的形容词
- 「我一定能胜任」「我会努力」—— 表决心没用，要证明你能做

❌ **不走心细节**：
- 到岗时间写「随时」（显得没规划）
- 实习时长写「看情况」（HR 要确定性）
- 公司名字写错、岗位名称不全

### 正文检查清单（生成后必须自查）

- [ ] 有没有提到具体的 JD 要求并回应？
- [ ] 匹配点是用事实证明，还是只说「我具备XX」？
- [ ] 到岗时间和实习时长写具体了吗？
- [ ] 有没有体现你研究过这个公司/岗位？（不是泛泛的「贵司很好」）
- [ ] 去掉了所有空洞形容词（「学习能力强」「沟通能力好」等）？
- [ ] 附件命名提到了吗？
- [ ] 读起来像是专门写给这家公司的，还是像批量模板？

### 示例对比

**❌ 差的写法（批量模板感）**：
> 尊敬的 HR：
>
> 您好！我是 XX 大学金融学大三学生，从贵司官网看到数据分析实习岗位，我对数据分析很感兴趣，希望能加入贵司学习。
>
> 我熟悉 Python 和 SQL，学习能力强，沟通能力好，能够快速适应工作。我相信我能胜任这份工作。
>
> 简历见附件，期待您的回复。

**✅ 好的写法（真诚且有针对性）**：
> XX 老师/HR 您好：
>
> 我是 XX 大学金融学大三学生，从官网看到贵司数据分析实习岗位。我之前在 XX 证券做过行研实习，用 Wind 处理过 30 家公司的财务数据，这次看到贵司岗位要求「熟悉金融数据处理」，感觉和我的经历很匹配。
>
> 关于 JD 要求，我想说明几点：
> - Python + SQL：我在课程项目中用 Python 处理过 10 万+条股票数据，写过自动化数据清洗脚本；SQL 能写多表关联和窗口函数
> - 实习时间：我可以每周 4 天（周一到周四），从 X 月持续到明年 1 月（期末考试周可能请假 3-5 天）
> - 我注意到贵司最近发布了 XX 行业报告，我之前实习时也跟过新能源赛道，对这个方向比较熟悉
>
> 简历见附件（已按要求命名为「姓名-学校-专业.pdf」），X 月 X 日可以到岗。期待您的回复！
