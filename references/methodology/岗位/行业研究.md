# 行业研究 / 投资研究专项

> 本文件只处理行研岗位。先执行《[简历方法论](../简历方法论.md)》，再叠加《[风格库](../风格库.md)》中的目标公司性质。结构参考 industry-resume-toolkit 的“企业性质 × 岗位”两层规则库，内容为本项目按行研场景原创整理。

## 1. 招聘方在验证什么

行研不是“搜资料 + 做报告”。招聘方要确认四件事：

1. 能不能界定研究问题和范围；
2. 能不能找到可信数据并处理口径；
3. 能不能从证据中形成判断，而不是堆材料；
4. 能不能把成果交付给真正使用它的人。

每段核心研究经历至少覆盖：**研究对象 → 数据与口径 → 分析方法/判断 → 产出及用途**。缺一项，就优先深挖，不先润色。

## 2. 同是行研，不同去向写法不同

| 目标场景 | 首要证明 | 优先展开的证据 | 不能混写 |
|---|---|---|---|
| 券商卖方研究 | 持续覆盖与稳定交付 | 覆盖公司/产业链、跟踪频率、数据源与口径、模型、纪要/日报/点评/深度报告、对团队研究的支撑 | 资料整理不等于独立覆盖；没有客户反馈就不写“获得市场认可” |
| 基金/资管买方研究 | 形成并验证投资判断 | 核心假设、催化剂、风险点、估值方法、跟踪验证、与组合讨论的关系 | 未参与决策就不写“推动建仓”；模拟收益不写成真实基金业绩 |
| 银行/金融机构研究或战略 | 专业口径、风险意识与决策支持 | 宏观/行业变量、资产负债或风险口径、监管边界、模型与报告用途 | 不把趋势判断写成确定预测，不省略风险边界 |
| 企业战略/产业研究 | 把外部信息转成经营判断 | 市场空间、产业链位置、竞争格局、客户/渠道/供给、关键发现、对进入/规划/资源配置的支持 | 不生搬证券估值话术；报告页数不能替代结论和用途 |
| 大厂战略/商业分析 | 用数据回答具体业务问题 | 用户/商家/产品场景、数据规模、竞品、关键指标、结论、建议与后续验证 | 没有访谈、问卷或实验就不能写；技术项目不能直接冒充用户研究 |
| 国央企产业研究 | 服务规划、政策理解与体系协同 | 政策口径、产业链环节、对标单位、阶段性成果、风险边界、协同对象 | 不用个人英雄叙事，不把方向性判断写成确定性承诺 |
| 外企/咨询研究 | 结构化解决问题并清楚表达 | 问题、假设/框架、证据、洞察、建议、影响；英文版用强动词和国际口径 | 不堆框架名，不把 PPT 页数当作唯一结果 |

使用这张表时，只选择一个主场景。目标同时具有两种属性时再叠加公司规则，例如“外资券商研究”以券商卖方为岗位骨架，再叠加外企的英文和表达规范。

## 3. 行业决定研究对象和指标

| 目标行业 | 应优先确认的内容 | 容易写错的地方 |
|---|---|---|
| 周期/制造/新能源 | 产业链位置、供需、产能、价格、成本、招标/订单、开工率、竞争格局、盈利弹性 | 只写宏观趋势，不落到供需和公司所在环节 |
| 消费/零售 | 品类、品牌、价格带、渠道、客群、动销、复购、门店/电商表现 | 用曝光代替经营结果，忽略产品和渠道 |
| TMT/互联网 | 用户与场景、产品、技术路线、生态、渗透率、活跃/留存/转化、商业化 | 把产品体验写成行业结论，把相关性写成因果 |
| 医疗健康 | 疾病领域、患者路径、治疗方案、临床阶段、监管、支付与商业化 | 医学术语不准；没有依据却下临床或疗效结论 |
| 金融 | 利率与流动性、资产负债、信用风险、监管、估值、业务结构 | 指标口径混用，只写收益不写风险 |

行业词必须来自用户确实研究过的对象、数据或指标。JD 中出现但事实库没有的专业词，只能标为缺口或发起深挖，不能直接写进简历。

## 4. 一段行研经历怎么组织

默认用 3–4 条 bullet，不给每条机械加“研究流程”“结构化处理”之类的泛化标签：

1. **覆盖与取数**：研究什么、覆盖多大范围、用哪些来源、如何校验口径；
2. **拆解与判断**：分析哪些变量、搭建什么框架/模型、形成什么类型的发现；
3. **交付与使用**：输出什么材料、多少份/多长周期、服务哪一步研究或决策；
4. **更新与验证（有则写）**：如何持续跟踪、复盘假设或修正模型。

bullet 仍遵守核心公式：**动作 + 落地方式 + 量化结果**。研究结论敏感或未公开时，可以写结论类型和研究用途，不泄露具体观点；没有明确业务结果时，优先量化覆盖范围、周期、模型、产出数量和采用情况。

## 5. 行研专用深挖

信息不足时，一轮只问最关键的 3–5 个：

1. 你具体研究了哪个行业、产业链环节或公司？覆盖了多久、多少对象？
2. 数据来自哪里？你处理过哪些口径冲突，怎么核验？
3. 你自己搭建或维护了什么模型、框架、数据库或跟踪表？
4. 你形成过什么判断？依据是什么，后来如何验证或修正？
5. 产出交给谁、用于什么环节？有无采用、反馈或后续动作？

## 6. 交付前排雷

- 报告页数、纪要篇数只是产出量，不等于研究价值；至少补上研究对象、方法或用途中的一项。
- 不把团队结论写成个人结论，不把协助整理写成独立覆盖。
- 不把竞赛、模拟盘或课程报告写成正式投研业绩。
- 估值方法、财务指标、产业链环节和医学术语必须准确；写进简历就默认能被追问三层。
- 不为了贴 JD 虚构访谈、专家交流、问卷、实地调研、可用性测试或投资决策。
