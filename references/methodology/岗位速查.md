## 投资银行 IBD

**HR 在验证**：项目经验真实性、财务建模能力、行业覆盖深度、加班抗压能力信号。

- **关键动作词**：协助完成（不要写「主导」「负责」）、搭建模型、撰写材料、尽调、对接、整理
- **必量化的维度**：参与项目数量和类型（IPO/再融资/并购）、涉及金额（亿元级）、模型行数/表数、材料页数
- **bullet 结构**：**项目类型与规模 → 你的具体分工 → 交付物 → 项目结果**
- **工具栈**：Excel 建模（三表/估值）、Wind/Bloomberg、PPT 制作
- **表述分寸**：实习生写「协助 MD/VP 完成」，不写「独立负责」；涉及保密信息用「某 A 股上市公司」替代真实名称
- **地雷**：夸大职责（实习生说自己「主导尽调」）；财务术语用错（EBITDA/FCF/WACC）；项目时间线与公开信息不符

## 风险管理 / 风控

**HR 在验证**：风险意识、数据敏感度、合规思维、应对异常的判断力。

- **关键动作词**：监测、识别、预警、审核、制定（规则/策略）、降低、控制
- **必量化的维度**：监测数据量、预警准确率/召回率、处置案件数、风险损失金额/下降幅度
- **bullet 结构**：**风险场景 → 监测/识别方式 → 处置动作 → 量化效果**
- **工具栈**：SQL、Python（风控模型）、规则引擎、反欺诈系统
- **表述分寸**：强调「合规」「及时」「准确」；涉及具体规则/阈值不写（商业机密）
- **地雷**：把「拒绝率」当成正面指标（拒太多会伤业务）；只写技术不写业务影响；把「风控」等同于「催收」

## 审计（四大 / 内资所）

**HR 在验证**：项目经历完整性、会计准则熟悉度、加班抗压能力、细节严谨度。

- **关键动作词**：执行（测试/程序）、核对、函证、盘点、编制（底稿/报告）、复核
- **必量化的维度**：参与项目数量和类型（年审/IPO/专项审计）、客户行业、测试样本量、底稿页数
- **bullet 结构**：**项目类型与客户规模 → 负责模块（科目/循环）→ 执行程序 → 发现问题或结论**
- **工具栈**：Excel、审计软件（鼎信诺/永拓/CaseWare）、Wind（上市公司数据）
- **表述分寸**：实习生写「在项目组指导下执行」，不写「独立完成审计」；涉及客户信息用「某制造业上市公司」替代
- **地雷**：夸大职责（实习生说「出具审计报告」）；会计科目/准则名称写错；项目时间与 busy season 不符

## 数据分析

**HR 在验证**：处理过的数据规模、工具真实熟练度、能否给出业务结论。

- **每段必有数据规模**：多少条 / 多少家 / 多少张表 / 多长周期
- **工具 3-5 个并标熟练度分档**，不做工具清单（Excel/SQL 是基本功不用单列，Python/Tableau/Power BI 等写具体库/功能）
- **bullet 结构**：**业务问题 → 方法与数据量 → 量化发现 → 业务影响**
- **结论挂业务指标**（收入/留存/转化率/GMV），不挂技术指标（准确率/F1/RMSE）
- **术语红线**（写了必被面试追问）：A/B 测试 ≠ 简单对比（要有显著性检验）；聚类 ≠ 分类；相关 ≠ 因果；机器学习模型要写清算法名称
- **学生加分**：课程项目包装成完整分析项目（有业务背景+数据来源+结论）、数据竞赛名次、爬虫/自动化的真实产出（定时爬取数据/自动化报表）
- **地雷**：只写工具不写数据量；只写过程不写结论；把探索性分析（EDA）说成「建模」

## 量化研究 / 量化策略

**HR 在验证**：编程能力、金融市场理解、因子挖掘思路、回测规范性、业绩真实性。

- **关键动作词**：构建、挖掘、回测、优化、跟踪、实现
- **必量化的维度**：因子数量、回测区间（年份）、样本股票池规模、策略收益/夏普/最大回撤、数据频率（日度/分钟级）
- **bullet 结构**：**策略类型 → 因子来源/逻辑 → 回测表现 → 改进措施**
- **工具栈**：Python（pandas/numpy/backtrader/zipline）、C++（高频）、Wind/Tushare/聚宽、数据库（MySQL/MongoDB）
- **业绩表述规范**：
  - 回测业绩：写清区间、基准、是否考虑交易成本
  - 实盘业绩：写清资金规模、策略类型、是否有风控
  - 竞赛成绩：写清平台（聚宽/优矿/米筐）、排名、参赛人数
- **表述分寸**：学生写「课程项目/竞赛项目」，不写「实盘策略」；回测结果不夸大（年化 50%+ 需要解释为什么这么高）
- **地雷**：只写收益不写回撤和夏普；回测未剔除未来函数（数据穿越）；把选股因子说成「择时策略」；把课程作业包装成「量化团队项目」

## 产品经理

**HR 在验证**：用户视角、需求定义能力、推动落地能力、数据 sense。

- **bullet 讲完整故事**：**用户/场景/痛点 → 方案设计 → 推动落地 → 业务指标变化**
- **最大地雷**：只讲功能不讲用户（写成了需求文档）
- **跨部门要具体**：「协调 N 人、X 周上线」，不写「沟通能力强」
- **方法论词**（用户画像、A/B 测试、优先级模型、KANO 模型）真用过才写，堆词无案例是负资产
- **工具栈**：Axure/Figma/Sketch（原型）、SQL/Excel（数据分析）、JIRA/Tapd（项目管理）
- **学生加分**：自己做出来的东西（小程序/公众号/用 AI 工具做的产品）比课程项目值钱；一份认真做的竞品分析/产品体验报告可以直接当项目写
- **地雷**：把「提需求」说成「产品设计」；只写上线不写数据变化；把课程大作业包装成「真实产品迭代」

## 运营（内容 / 用户 / 活动）

**HR 在验证**：操盘过的真实项目、数据结果、资源调动能力、用户感知力。

- **关键动作词**：策划、执行、推广、维护、转化、增长、拉新、促活、留存
- **必量化的维度**：用户规模（参与人数/覆盖量）、增长数据（新增/留存率/转化率）、内容产出量（篇数/播放量/互动量）、活动 ROI
- **bullet 结构**：**目标（拉新/促活/转化）→ 方案（渠道+玩法）→ 执行（资源+周期）→ 结果（数据）**
- **工具栈**：数据分析（SQL/Excel/增长看板）、内容工具（剪映/PS/AI 工具）、推广渠道（公众号/小红书/抖音）
- **表述分寸**：学生可以写校园活动/社团运营/个人账号，但要写清「从 0 到 1」还是「接手后优化」
- **地雷**：只写做了什么不写数据；把「发了 10 篇文章」当成成果（没有阅读量/转化数据）；虚报数据（面试会追问渠道和细节）

## 市场营销

**HR 在验证**：Campaign 操盘经验、品牌感知力、渠道资源、ROI 意识。

- **关键动作词**：策划、执行、投放、合作、传播、曝光、转化
- **必量化的维度**：Campaign 预算与 ROI、曝光量/触达人数、转化数据（线索/注册/购买）、渠道数量与类型
- **bullet 结构**：**Campaign 目标 → 创意/渠道策略 → 执行与资源 → 曝光与转化数据**
- **工具栈**：投放平台（巨量引擎/广点通/小红书蒲公英）、数据分析（GA/神策/GrowingIO）、设计工具（Canva/PS）
- **学生加分**：真实投放经验（哪怕预算只有几百块）、品牌 Campaign 复盘作品、KOL/社群资源
- **地雷**：只写创意不写数据；把「转发了海报」说成「市场推广」；虚报预算和 ROI（面试会追问投放明细）

## 战略 / 咨询

**HR 在验证**：结构化思维、商业洞察、PPT 制作能力、项目经验真实性。

- **关键动作词**：分析、梳理、访谈、研究、建议、输出、支持
- **必量化的维度**：项目数量与类型（战略规划/尽调/行研）、访谈对象数量、研究范围（行业/公司数）、交付物页数
- **bullet 结构**：**项目背景与目标 → 分析框架/方法 → 关键发现 → 交付物与影响**
- **工具栈**：Excel（财务建模）、PPT（咨询级排版）、行业数据库（Wind/iResearch/Euromonitor）
- **表述分寸**：实习生写「在项目组支持下完成」，不写「独立主导」；涉及客户信息脱敏
- **地雷**：把「做了 PPT」说成「战略咨询」；只写框架不写洞察（「用了波特五力分析」≠ 有价值）；虚构访谈对象或行业数据

## 财务

**HR 在验证**：会计准则熟悉度、财务分析能力、系统操作熟练度、细节严谨度。

- **关键动作词**：核算、编制、审核、分析、预测、对账、报税
- **必量化的维度**：处理凭证/发票数量、报表类型与周期（月报/季报/年报）、预算金额、成本节约金额
- **bullet 结构**：**财务模块 → 具体工作 → 处理量/金额 → 发现问题或优化结果**
- **工具栈**：ERP 系统（SAP/用友/金蝶）、Excel（函数/透视表）、报税系统
- **证书加分**：CPA（已过 N 科）、CFA（已过 Level X）、初级/中级会计职称
- **地雷**：会计科目名称写错；把「录凭证」包装成「财务分析」；虚报处理金额（和公司规模不匹配会穿帮）

---

## 销售 / BD

**HR 在验证**：业绩真实性、客户资源、谈判能力、结果导向。

- **关键动作词**：开发、维护、拓展、谈判、签约、达成、推进
- **必量化的维度**：客户数量、签约金额/订单数、业绩目标完成率、客户续约率/复购率
- **bullet 结构**：**客户类型 → 开发/维护动作 → 成交结果（金额/数量）→ 超额完成或客户续约**
- **工具栈**：CRM 系统（Salesforce/纷享销客）、Excel（销售漏斗分析）、行业资源平台
- **学生加分**：校园推广经验（卖课/卖货/拉赞助）、社团外联资源、真实成交案例（哪怕金额小）
- **地雷**：只写客户数不写成交额；虚报业绩（面试会追问客户名称和合同细节）；把「发了 100 封开发信」当成成果（没有转化数据）

## 使用说明

1. **找到对应岗位章节**：直接用岗位名搜索，或找最接近的岗位类型
2. **套用 bullet 结构**：每段经历按对应的「bullet 结构」重写
3. **检查术语和量化**：确保关键动作词、必量化维度都覆盖到
4. **避开地雷**：对照「地雷」清单自查，面试追问三层不露馅

**补充新岗位**：遇到未覆盖的岗位（如采购、客服、供应链等），可参考相近岗位的框架，或在投递过程中从 JD 提取术语后补充进来。
