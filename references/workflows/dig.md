# 深挖工作流（dig）

> 触发：用户想补某段经历的细节，或某段经历太薄需要充实。
> 目标：把某段经历的「待挖掘」问透，沉淀成可直接进简历的事实和表述。

路径统一按根目录 `SKILL.md` 的 `SKILL_ROOT` / `CAREER_ROOT` 约定解析。

## 1. 选经历

- 用户点名了某段经历 → 在 `profile/experiences/` 里模糊匹配文件名和标签
- 没点名 → 读 `profile/_index.md`，列出完整度为「低/中」的经历及其价值（对哪些目标岗位有用）

**交互方式**：
- 支持结构化选择的 agent：用当前界面能力让用户点选经历
- 不支持的 agent：列出经历编号和一句话描述，让用户回复编号或经历名称

## 2. 提问（迭代式，不是问卷式）

读该经历文件的「事实清单」和「待挖掘」，按方法论深挖六问组织问题，具体追问角度用 `references/methodology/挖掘问题库.md` A 节（按经历类型选）：

- 一轮只问 3-5 个，优先问「待挖掘」里已列出的
- 用户答完再追问下一轮，直到：用户喊停 / 六维度都有着落 / 连续两轮没新信息
- 只问事实。动机、规划、感受是面试问题，不进简历也不问

## 3. 沉淀

1. 新事实逐条写入「事实清单」（含数字、来源=本人口述及日期）
2. 把最有价值的素材打磨成 1-3 条新「现成表述」bullet（动作+落地+量化），按角度归组
3. 删掉已回答的「待挖掘」条目，补充新发现的空洞
4. 完整度提升时更新 frontmatter 和 `_index.md`

**格式参考**：`profile/experiences/_TEMPLATE.md`

## 4. 汇报

本次挖到的最硬的 2-3 个事实、新增的现成表述、该经历现在适配哪些岗位方向。
