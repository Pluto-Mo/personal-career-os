# 投递工作流（apply）

> 触发：用户粘贴 JD、发职位链接、或说要投某个岗位。
> 目标：产出「复制粘贴就能投」的全套材料。

## 0. 拿到 JD

- 用户消息里有 JD 文本 → 直接用；给的是文件路径 → 读文件
- 都没有 → 请用户把 JD 完整粘贴过来（公司名、岗位名一起）

## 1. 读上下文（顺序固定）

1. `methodology/简历方法论.md` —— 全程遵守
2. `profile/_index.md` → `basics.md`、`preferences.md`、`links.md`
3. 按 JD 方向挑出相关经历，读对应 `profile/experiences/*.md`
4. 按目标企业性质读 `methodology/风格库.md` 对应节；按岗位读 `methodology/岗位速查.md` 对应节（没有对应节就现场研究该岗位话语体系，并把结论沉淀成新的一节）

注意 `preferences.md` 中带时效的信息（到岗时间等），过期就先问用户更新。

## 2. JD 分析（展示给用户）

按方法论的 JD 分析框架输出：硬性门槛比对（逐条 ✓/✗）、高频关键词、加分项、风格信号、**文件命名要求**（如有）。

**文件命名要求提取**：
- JD 中明确要求的命名格式（如「姓名-学校-专业.pdf」「学校-姓名-岗位」等）
- 没有要求时，默认用 `城市-学校-姓名-专业-年级`
- 第一次遇到某公司的命名要求时，记入 `methodology/岗位速查.md` 对应公司章节（方便下次投递同公司其他岗位时直接调用）

硬性门槛有 ✗ 时明确指出风险，让用户决定是否继续。

## 3. 经历选配（和用户确认后再动笔）

给出方案并等用户确认：

- 选哪几段经历、排什么顺序、每段主打什么角度（引用经历文件里的「现成表述」角度）
- 舍弃/压缩哪些、为什么
- 若关键经历「完整度：低/中」且 JD 高度相关：现场按方法论「深挖六问」问 3-5 个事实问题（答案回写经历文件），或建议先走深挖工作流
- 涉及【待确认】标记的事实（如证书表述、专业口径）第一次用到时必须问清，确认后更新 profile

用户确认后才进入生成。用户如说「直接出」，跳过讨论但【待确认】事实仍必须问。

## 4. 生成简历

1. 建目录 `applications/YYYY-MM-DD-公司-岗位/`（当天日期）
2. 复制 `template/resume.html` 进去，只改内容不改版式骨架
3. 改写规则：方法论三步（排雷 → 价值翻译 → 量化），关键词对齐 JD，风格对齐企业性质
4. **保真**：每个数字都能在经历文件事实清单里找到；缺的问用户，没确认就不写
5. JD 有硬性时间要求时，在技能区下加一行「补充信息」（到岗时间、每周天数，从 preferences.md 取）
6. links.md 中「适用场景」匹配的链接写进简历对应位置

## 5. 渲染与一页校验

- Windows：`pwsh scripts/render.ps1 -Html "applications/<目录>/resume.html" -Name "<简历文件名>"`
- macOS / Linux：`bash scripts/render.sh "applications/<目录>/resume.html" "<简历文件名>"`

**文件名规则**（优先级从高到低）：
1. **JD 明确要求的格式优先**（如「姓名-学校-专业」「学校+姓名+应聘岗位」等）—— 从第 2 步提取的命名要求
2. 没有要求时，默认 `城市-学校-姓名-专业-年级`（从 preferences/basics 取）
3. 命名中的空格和特殊字符统一替换为 `-`

- 超页 → 按方法论删减顺序删内容重渲，直到通过。禁止把字号降到 10pt 以下
- 渲染通过后用 Read 看一眼 PNG，确认排版没有溢出、错行

## 6. 生成配套材料（同目录）

- `jd.md`：JD 原文 + 第 2 步的分析
- `email.md`：按方法论邮件规范写好 **主题** 和 **正文**（直接可复制），末尾列附件清单
- `notes.md`：用了哪些经历/角度、关键词覆盖情况、本次讨论中的遗留问题

## 7. 记账与沉淀（不能跳过）

1. 在 `applications/_log.md` 追加一行（日期/公司/岗位/渠道/状态=已准备）
2. **回写**：本次对话中用户确认的新事实 → 写进对应经历文件的事实清单，消掉已回答的「待挖掘」；涉及完整度变化就更新 `_index.md`
3. 全新经历 → 提示用户之后走沉淀工作流补录

## 8. 汇报

最终消息里给出：产物文件清单（可点击路径）、3 个关键定制点、复制粘贴指引（邮件主题贴哪、正文贴哪、附件是哪个文件）。
